## Prerequisites

Before you begin, make sure you have the following installed:

- **Docker** (version 20.10 or higher)
- **Docker Compose** (version 2.0 or higher)
- **Git** (for cloning the repository)

Alternatively, if running without Docker:
- **Java 21** or higher
- **Maven 3.8+**
- **PostgreSQL 14+**

<Note>
  The easiest way to get started is using Docker Compose, which handles all dependencies automatically.
</Note>

## Quick Start with Docker

<Steps>
  <Step title="Clone the Repository">
    Clone the Furniture API repository to your local machine:
    
    ```bash
    git clone <repository-url>
    cd furniture-api
    ```
  </Step>

  <Step title="Start the Application">
    Launch the API and all dependencies using Docker Compose:
    
    ```bash
    docker compose up
    ```
    
    This command will:
    - Build the Furniture API Docker image
    - Start the PostgreSQL database
    - Initialize the database schema
    - Launch the API service on port 8082
    
    <Info>
      The first run may take a few minutes as Docker downloads images and builds the application.
    </Info>
  </Step>

  <Step title="Verify the API is Running">
    Once the containers are up, verify the API is running:
    
    ```bash
    curl http://localhost:8082/actuator/health
    ```
    
    You should see a response like:
    
    ```json
    {
      "status": "UP"
    }
    ```
  </Step>

  <Step title="Access Swagger UI">
    Open your browser and navigate to the interactive API documentation:
    
    ```
    http://localhost:8082/doc/swagger-ui.html
    ```
    
    The Swagger UI provides:
    - Complete API documentation
    - Interactive endpoint testing
    - Request/response examples
    - Schema definitions
  </Step>
</Steps>

## Alternative: Running with Maven

If you prefer to run the application without Docker:

<Steps>
  <Step title="Configure Database">
    Ensure PostgreSQL is running and create a database. Update the `src/main/resources/application.properties` file with your database credentials:
    
    ```properties
    spring.datasource.url=jdbc:postgresql://localhost:5432/furniture_db
    spring.datasource.username=your_username
    spring.datasource.password=your_password
    ```
  </Step>

  <Step title="Build and Run">
    Build and start the application using Maven:
    
    ```bash
    ./mvnw clean install
    ./mvnw spring-boot:run
    ```
    
    The API will start on port 8082.
  </Step>
</Steps>

## Making Your First API Request

Let's create a product category and then add a product:

<CodeGroup>

```bash cURL
# Create a category
curl -X POST http://localhost:8082/api/v1/categories \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Living Room",
    "description": "Furniture for living spaces",
    "is_active": true
  }'

# Create a product
curl -X POST http://localhost:8082/api/v1/products \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Modern Sofa",
    "description": "Comfortable 3-seater sofa with fabric upholstery",
    "sku": "SOFA-001",
    "category_id": 1,
    "price": 899.99,
    "cost_price": 450.00,
    "weight_kg": 85.5,
    "is_active": true
  }'

# Get all products
curl http://localhost:8082/api/v1/products
```

```javascript JavaScript
// Create a category
const categoryResponse = await fetch('http://localhost:8082/api/v1/categories', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'Living Room',
    description: 'Furniture for living spaces',
    is_active: true
  })
});

const category = await categoryResponse.json();

// Create a product
const productResponse = await fetch('http://localhost:8082/api/v1/products', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'Modern Sofa',
    description: 'Comfortable 3-seater sofa with fabric upholstery',
    sku: 'SOFA-001',
    category_id: category.id,
    price: 899.99,
    cost_price: 450.00,
    weight_kg: 85.5,
    is_active: true
  })
});

const product = await productResponse.json();
console.log('Created product:', product);

// Get all products
const productsResponse = await fetch('http://localhost:8082/api/v1/products');
const products = await productsResponse.json();
console.log('All products:', products);
```

```python Python
import requests

base_url = "http://localhost:8082/api/v1"

# Create a category
category_data = {
    "name": "Living Room",
    "description": "Furniture for living spaces",
    "is_active": True
}

category_response = requests.post(
    f"{base_url}/categories",
    json=category_data
)
category = category_response.json()

# Create a product
product_data = {
    "name": "Modern Sofa",
    "description": "Comfortable 3-seater sofa with fabric upholstery",
    "sku": "SOFA-001",
    "category_id": category["id"],
    "price": 899.99,
    "cost_price": 450.00,
    "weight_kg": 85.5,
    "is_active": True
}

product_response = requests.post(
    f"{base_url}/products",
    json=product_data
)
product = product_response.json()

print(f"Created product: {product}")

# Get all products
products_response = requests.get(f"{base_url}/products")
products = products_response.json()
print(f"All products: {products}")
```

</CodeGroup>

## Understanding the Response

When you create a product, the API returns a `ProductResponse` object:

```json
{
  "id": 1,
  "name": "Modern Sofa",
  "description": "Comfortable 3-seater sofa with fabric upholstery",
  "sku": "SOFA-001",
  "category_id": 1,
  "price": 899.99,
  "cost_price": 450.00,
  "weight_kg": 85.5,
  "is_active": true,
  "created_at": "2026-03-03T10:30:00",
  "product_dimension_id": null,
  "product_inventory_id": null,
  "image_ids": []
}
```

<Note>
  All property names use snake_case formatting in JSON responses, as configured in the application properties.
</Note>

## Common Operations

### Search Products

<CodeGroup>

```bash Get by SKU
curl http://localhost:8082/api/v1/products/search/sku/SOFA-001
```

```bash Search by Name
curl http://localhost:8082/api/v1/products/search/name/sofa
```

```bash Filter by Price Range
curl "http://localhost:8082/api/v1/products/price-range?minPrice=500&maxPrice=1000"
```

```bash Get by Category
curl http://localhost:8082/api/v1/products/category/1
```

```bash Pagination
curl "http://localhost:8082/api/v1/products/pagination?page=0&pageSize=10"
```

</CodeGroup>

### Manage Inventory

<CodeGroup>

```bash Check Low Stock
curl http://localhost:8082/api/v1/inventory/low-stock
```

```bash Get Out of Stock Items
curl http://localhost:8082/api/v1/inventory/out-of-stock
```

```bash Get Product Inventory
curl http://localhost:8082/api/v1/inventory/product/1
```

```bash Decrease Stock
curl -X PUT "http://localhost:8082/api/v1/inventory/decrease-stock?productId=1&quantity=5"
```

</CodeGroup>

## Configuration

The API is configured via `application.properties`. Key settings:

```properties
# Server Configuration
spring.application.name=microservice-rest
server.port=8082

# Swagger UI
springdoc.swagger-ui.path=/doc/swagger-ui.html

# JSON Response Format
spring.jackson.property-naming-strategy=SNAKE_CASE
```

## Stopping the Application

To stop the Docker containers:

```bash
docker compose down
```

To also remove volumes (including database data):

```bash
docker compose down -v
```

## Health Checks

The API includes Spring Boot Actuator endpoints for monitoring:

```bash
# Health check
curl http://localhost:8082/actuator/health

# Application info
curl http://localhost:8082/actuator/info
```
